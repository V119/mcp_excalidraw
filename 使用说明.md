## 架构

```
┌──────────────────────────────────────────────────────┐
│  Claude Code                                          │
│  └─ MCP Server (docker run -i --rm)                  │
│       │  stdin/stdout (MCP 协议)                      │
│       │                                               │
│       └─ HTTP ──────────────────────────────┐        │
│                                              │        │
│  Canvas Server (Docker 后台运行)              │        │
│  ├─ Web UI: http://localhost:3000            │        │
│  ├─ REST API: http://localhost:3000/api/     │        │
│  └─ WebSocket 实时同步                       │        │
└──────────────────────────────────────────────────────┘
```

- **Canvas Server** (`mcp-excalidraw-canvas`)：后台容器，提供可视化画布 + REST API，端口 3000
- **MCP Server** (`mcp-excalidraw`)：Claude Code 按需拉起，通过 stdin/stdout 通信，通过 HTTP 与 Canvas 同步状态

## 快速启动

### 启动 Canvas 服务器

```bash
cd /Users/haoyang/Documents/中立/mcp/mcp_excalidraw

```

### 停止 Canvas 服务器

```bash
bash start.sh stop
```

### 重启 Canvas 服务器

```bash
bash start.sh restart
```

### 查看状态

```bash
bash start.sh status
```

## Claude Code 配置（子 Agent 模式）

为减少主对话上下文中 26 个 MCP 工具的 schema 占用，MCP 服务器**不在全局加载**，而是通过专用子 Agent 按需加载。

### 配置文件

- **Agent 定义**：`~/.claude/agents/excalidraw-diagrammer.md`
- **Skill 文件**：`~/.claude/skills/excalidraw-skill/`
- **全局 settings**：`~/.claude/settings.json`（不含 MCP 配置）

### 使用方式

**方式一（推荐）：专用 Agent 会话**

在终端中启动独立的 Claude Code 会话，仅加载 Excalidraw 工具：

```bash
claude --agent excalidraw-diagrammer
```

此会话中只有 Excalidraw 的 26 个 MCP 工具，不消耗主对话上下文。适合绘制复杂图表。

**方式二：主对话中委派子任务**

在主对话中直接描述图表需求（如「画一个微服务架构图」），Claude 会自动通过 skill 引导，使用 Bash + curl 调用 Canvas REST API 完成画图。此方式无需 MCP 工具，通过 HTTP 端点完成全部操作。

（注：MCP 子 Agent 委派存在已知 bug #30280，使用 REST API fallback 更可靠。）

### Agent 内 MCP 配置（在 agent 文件中定义）

```json
{
  "mcpServers": {
    "mcp_excalidraw": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "EXPRESS_SERVER_URL=http://host.docker.internal:3000",
        "-e", "ENABLE_CANVAS_SYNC=true",
        "mcp-excalidraw:latest"
      ]
    }
  }
}
```

MCP 服务器仅在 `claude --agent excalidraw-diagrammer` 时启动。

## Skill 说明

项目中 `skills/excalidraw-skill/` 已安装到 `~/.claude/skills/excalidraw-skill/`，为 AI 提供以下指导：

Claude Code 在需要绘制图表时会自动调用该 skill，无需手动操作。

## 常见问题

**Q: 重启 Claude Code 后 MCP 未连接？**
确保 Canvas 容器在运行（`bash start.sh status`），然后在 Claude Code 中执行 `/mcp` 检查。

**Q: 画布上的更改没有同步？**
MCP 服务器通过 `EXPRESS_SERVER_URL=http://host.docker.internal:3000` 连接 Canvas，确保容器端口映射正常。

**Q: 如何彻底删除并重建？**
```bash
bash start.sh stop
docker rm -f mcp-excalidraw-canvas 2>/dev/null
docker rmi mcp-excalidraw-canvas:latest mcp-excalidraw:latest
# 重新加载镜像
docker load -i docker-images/mcp-excalidraw-canvas.tar
docker load -i docker-images/mcp-excalidraw-mcp.tar
bash start.sh start
```
